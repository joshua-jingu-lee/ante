# 개요

Treasury는 Ante의 **중앙 자금(예산) 관리 모듈**이다.
**계좌(Account)별로 인스턴스가 생성**되며, 각 계좌의 잔고를 소유하고, 봇별로 예산을 배분하며, 예산 범위 내에서 거래가 이루어지도록 관리한다.
회사의 "자금팀(Treasury)"과 동일한 역할로, 예산 할당/회수/예약만 담당한다.

> **포지션은 Trade 모듈이 단일 소유자**이다. Treasury는 포지션을 추적하지 않는다.
> 미실현 손익 기반 리스크 모니터링(평가손실 한도 등)은 **Rule Engine**의 영역이다.

**주요 기능**:
- **중앙 자금 관리**: 전체 계좌 잔고 관리
- **봇별 예산 배분**: 봇에 자금 할당/회수
- **자금 예약**: 주문 시 자금 예약, 체결/취소 시 정산
- **예산 여력 판단**: 주문 승인 시 "남은 예산 ≥ 주문 금액" 검증
- **EventBus 연동**: 주문 체결/취소 이벤트 처리로 자금 정산
