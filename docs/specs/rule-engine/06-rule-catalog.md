# 구체적 룰 목록

### 계좌 룰 (Account Rules)

> 소스: `src/ante/rule/global_rules.py`

계좌 레벨에서 해당 계좌의 모든 봇에 적용되는 룰들.

### 전략별 룰 (Strategy Rules)

> 소스: `src/ante/rule/strategy_rules.py`

특정 봇/전략에만 적용되는 룰들.

| 룰 이름 | 계층 | 설명 | 핵심 파라미터 | 시나리오 예시 |
|---------|------|------|--------------|-------------|
| `DailyLossLimitRule` | 계좌 | 일일 손실 한도 초과 시 추가 매수 차단 (매도는 허용) | `max_daily_loss_percent` (기본 5%) | 시장 급락으로 모든 전략이 손실 진입. 일일 손실이 5%에 도달하자 추가 매수가 거부되고 사용자에게 알림이 발송된다. 매도(손절)는 허용하여 포지션 정리 경로를 보장한다 |
| `TotalExposureLimitRule` | 계좌 | 총 포지션 노출 한도 초과 시 추가 매수 차단 (매도는 허용) | `max_exposure_percent` (기본 20%), `max_exposure_amount` | Agent가 강세 신호를 감지하여 여러 종목에 공격적으로 매수 시도. 총 노출이 자산의 20%에 도달하자 추가 매수가 거부되고 사용자에게 알림이 발송된다. 매도(포지션 축소)는 허용한다 |
| `TradingHoursRule` | 계좌 | 거래 허용 시간 외 거래 차단 | Account의 `trading_hours_start`/`trading_hours_end`, `timezone`에서 자동 주입 | Agent가 장 마감 후 뉴스를 분석하여 즉시 매수 신호 발생. 장외 시간이므로 주문이 차단되어 다음 날 정규장에서 재판단 가능 |
| `PositionSizeRule` | 전략별 | 종목별 포지션 사이즈 제한 | `max_position_percent` (기본 10%), `max_position_amount` | 모멘텀 전략이 삼성전자에 확신을 갖고 전액 매수 시도. 종목당 10% 한도에 걸려 포지션이 제한되어 단일 종목 리스크 억제 |
| `UnrealizedLossLimitRule` | 전략별 | 봇의 미실현 손실 한도 초과 시 추가 매수 차단 (매도는 허용) | `max_unrealized_loss_percent` (기본 10%) | 역추세 전략이 하락 중인 종목을 물타기 매수 시도. 이미 미실현 손실이 -11%이므로 추가 매수가 차단되고, 손절 매도는 허용되어 탈출 경로 유지 |
| `TradeFrequencyRule` | 전략별 | 단위 시간당 거래 빈도 제한 | `max_trades_per_hour` (기본 10) | 변동성 폭발 구간에서 스캘핑 전략이 분당 수 회 매매 시도. 시간당 10회 한도에 도달하자 추가 주문이 차단되어 과잉 거래에 의한 수수료 손실 방지 |

> **용어 산식**:
> - **일일 손실률** = `당일 실현 손익 / 전일 총 자산 × 100`
> - **총 노출(Exposure)** = `전 봇의 보유 포지션 평가액 합계`. 총 노출률 = `총 노출 / 총 자산 × 100`
> - **포지션 사이즈** = `해당 종목 보유액 + 신규 주문 금액`. 포지션 비율 = `포지션 사이즈 / 봇 할당 예산 × 100`
> - **미실현 손실률** = `(현재 평가액 - 매입 금액) / 봇 할당 예산 × 100` (음수면 손실)

> **설계 의도 (UnrealizedLossLimitRule)**: 미실현 손익 기반 리스크 모니터링은 Treasury가 아닌 Rule Engine에서 담당한다.
> 이를 통해 공격적 전략에는 넓은 손실 허용(-20%), 보수적 전략에는 좁은 허용(-5%)을 전략별로 설정할 수 있어 AI 전략의 자유도를 극대화한다.
