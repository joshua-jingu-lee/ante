# 2계층 평가 흐름

주문 요청이 들어오면 해당 계좌의 RuleEngine이 다음 순서로 평가합니다:

1. **계좌 룰 평가**: 계좌 레벨 룰을 우선순위 순서대로 평가. BLOCK/REJECT 발생 시 즉시 거부.
2. **전략별 룰 평가**: 해당 봇/전략에 바인딩된 룰을 우선순위 순서대로 평가.
3. **결과 통합**: 모든 평가 결과를 종합하여 최종 판정. BLOCK/REJECT은 즉시 중단하며, WARN은 PASS보다 우선.
4. **조치 실행**: BLOCK/REJECT 시 알림 발송, 봇 중지, 계좌 중지 등 RuleAction에 따른 후속 조치.

전략별 룰은 계좌 룰을 위반할 수 없으며, 계좌 룰에서 BLOCK/REJECT된 주문은 전략별 룰 평가 없이 즉시 거부됩니다.

### 룰 충돌 해결 규칙

- **계층 우선**: 계좌 룰이 전략별 룰보다 항상 먼저 평가된다. 전략별 룰은 계좌 룰의 판정을 완화하거나 우회할 수 없다.
- **우선순위 순서**: 같은 계층 내에서는 낮은 `priority` 값이 먼저 평가된다.
- **즉시 중단**: BLOCK 또는 REJECT 결과가 발생하면 이후 룰 평가를 스킵하고 즉시 거부한다.
- **결과 우선순위**: `BLOCK > REJECT > WARN > PASS`. 모든 룰이 통과한 경우에만 PASS이며, WARN이 하나라도 있으면 최종 결과는 WARN(통과하되 로깅).
