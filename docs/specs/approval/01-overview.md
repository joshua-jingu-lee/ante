# Approval 모듈 세부 설계 - 개요

# 개요

Approval 모듈은 **AI Agent의 요청을 사용자가 승인·거절하는 의사결정 체계**다.
Ante의 정체성에서 사용자는 대표, Agent는 직원이다. 직원이 업무 변경을 원하면 대표에게 결재를 올리고, 대표가 승인하면 즉시 반영된다.

**핵심 원칙:**
- Agent는 요청만 생성할 수 있다. 실행 권한은 사용자의 승인 후에만 부여된다.
- 사용자는 Dashboard에서 결재 없이 동일 작업을 직접 수행할 수 있다 (대표 권한).
- 승인된 요청은 시스템이 즉시 자동 실행한다.

**주요 기능:**
- **결재 요청 생성**: Agent가 CLI를 통해 변경 요청을 제출
- **결재 목록 관리**: 대기·승인·거절·보류·만료 상태 추적
- **자동 실행**: 승인 시 요청 유형에 따라 대상 모듈의 동작을 즉시 호출
- **다채널 알림 및 응답**: 요청 생성 시 외부 메신저로 통보하고, 사용자는 같은 채널에서 바로 승인·거절할 수 있다

**결재 처리 경로:**

사용자는 세 가지 채널에서 결재를 처리할 수 있다. 어떤 경로든 `ApprovalService.approve()` / `reject()`를 호출하는 것은 동일하다.

```
                    ┌─────────────┐
                    │ Agent       │
                    │ (CLI 요청)  │
                    └──────┬──────┘
                           │ create
                           ▼
                  ┌─────────────────┐
                  │ ApprovalService │
                  └────────┬────────┘
                           │ ApprovalCreatedEvent
                           ▼
              ┌────────────────────────┐
              │ Notification           │
              │ (Telegram 등)          │
              └────────────────────────┘
                           │
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
    ┌──────────┐   ┌──────────────┐  ┌──────────┐
    │Dashboard │   │ 외부 메신저   │  │ CLI      │
    │ (Web)    │   │ (Telegram)   │  │          │
    └────┬─────┘   └──────┬───────┘  └────┬─────┘
         │                │               │
         └────────────────┼───────────────┘
                          ▼
                 approve() / reject()
                          ▼
                    (자동 실행)
```
