# D-005: EventBus 적용 범위 (2026-03-12)

> Ante 설계 결정 기록.
> 인덱스: [README.md](README.md)

**결정**: 핵심 흐름만 EventBus, 나머지는 직접 호출 (하이브리드)

**EventBus 대상** (1:N 브로드캐스트 성격):
- 주문 흐름 (OrderRequest, OrderFilled 등)
- 시스템 이벤트 (BotStarted, BotStopped, BotError)
- 알림 트리거 (NotificationEvent)
- 백테스트 완료 (BacktestCompleteEvent)

**직접 호출 대상** (1:1 요청-응답 성격):
- 데이터 조회
- 자금 확인
- 설정 읽기

**근거**:
- 이벤트는 "발생 사실을 여러 곳에 알리는" 패턴에 적합
- 데이터 조회 같은 요청-응답은 직접 호출이 자연스러움
- 1인 개발에서 전면 이벤트는 생산성 저하
- 필요 시 직접 호출을 이벤트로 전환하는 것은 용이

**미결 사항**:
- 구체적 항목별 구분은 세부 설계 시 확정
