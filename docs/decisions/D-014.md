# D-014: 설정 API 구조 — Config API 단일화 (2026-03-23)

> Ante 설계 결정 기록.
> 인덱스: [README.md](README.md)

**결정**: 대시보드 설정 화면의 모든 설정 조회·변경을 기존 Config API(`GET/PUT /api/config/{key}`)로 커버한다. 별도 `/api/settings` 엔드포인트를 만들지 않는다.

**근거**:
- 이미 동적 Config 시스템(`ConfigStore` + `ConfigChangedEvent`)이 구현되어 있음
- 알림 설정, 시스템 설정 등 대시보드 설정 항목은 모두 `config` 테이블의 key-value로 표현 가능
- 별도 설정 API를 만들면 Config API와 책임이 중복되어 혼란 유발
- `ConfigChangedEvent`를 통한 런타임 반영 메커니즘을 그대로 활용 가능

**영향 범위**:
- `docs/dashboard/user-stories/settings.md`: API 경로를 `PUT /api/config/{key}`로 수정 완료
- 프론트엔드: 설정 화면에서 Config API를 직접 호출
