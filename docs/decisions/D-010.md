# D-010: Agent 친화적 설계 원칙 (2026-03-12)

> Ante 설계 결정 기록.
> 인덱스: [README.md](README.md)

**결정**: 운용 Agent가 시스템을 독립적으로 발견·학습·사용할 수 있도록 설계

**구성 요소**:

1. **AGENT.md (운용 Agent 온보딩 가이드, 초기 구상)**
   - 프로젝트 루트에 배치
   - 시스템 요약, Agent 역할/권한 범위, CLI 워크플로우, 전략 작성 규약, 리포트 포맷
   - AGENTS.md(개발 Agent용)와 역할 분리

2. **CLI 자기 설명 (Self-documenting)**
   - 모든 커맨드에 상세 `--help` 제공
   - Agent가 `ante --help`만으로 사용 가능한 기능 파악 가능

3. **전략 템플릿 & 예제**
   - `strategies/_template.py` — 최소 뼈대
   - `strategies/_examples/` — 참조용 예제 전략

4. **데이터 카탈로그 CLI**
   - `ante data list` — 보유 데이터 목록 (종목, 타입, 기간, 건수)
   - `ante data schema <type>` — 컬럼 정의 확인

5. **구조화된 출력**
   - 모든 CLI 커맨드에 `--format json` 옵션 지원
   - Agent가 결과를 파싱하여 다음 행동을 결정할 수 있도록

6. **리포트 스키마 명세**
   - `ante report schema` — 제출 포맷 조회
   - 초기 구상에서는 AGENT.md에도 포맷 문서화

**후속 상태**:
- 별도 `AGENT.md`는 유지하지 않고 관련 내용은 `AGENTS.md`, 전략 에이전트 정의, CLI self-documentation으로 흡수

**근거**:
- AI Agent가 사람의 도움 없이 시스템을 발견 → 학습 → 사용하는 전체 루프가 자연스러워야 함
- CLI self-doc + 구조화된 출력으로 Agent의 탐색-실행-파싱 사이클 지원
- 템플릿/예제가 있어야 Agent가 올바른 전략 파일을 생성할 수 있음
