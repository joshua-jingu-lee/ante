# D-006: DB 엔진 및 데이터 저장 전략 (2026-03-12)

> Ante 설계 결정 기록.
> 인덱스: [README.md](README.md)

**결정**: SQLite (WAL 모드) + Parquet 이원 구조

**저장소 역할 분담**:
- **Parquet**: 시세 데이터 (OHLCV 일봉/분봉 등) — 봇 운영 중 수집 데이터 포함
- **SQLite**: 운영 데이터 (주문/체결 이력, 봇/전략 설정, 자금/포지션 상태, 전략 리포트)

**근거**:
- 시세 데이터는 Parquet으로 분리되어 DB 부담이 작음 → 경량 DB로 충분
- 단일 asyncio 프로세스 아키텍처에서 동시 쓰기 부담 없음 → SQLite의 동시성 제한이 문제되지 않음
- 별도 서버 프로세스 불필요 → N100 환경에서 리소스 절약
- 파일 기반으로 백업/이동 단순
- Python 표준 라이브러리 포함 (의존성 제로)
- 향후 필요 시 ORM 레이어를 통해 PostgreSQL 전환 가능
