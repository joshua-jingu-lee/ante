# 결재함

> ⚠️ 이 문서는 이 화면을 이용하는 사용자의 시나리오와 행동에 따른 제약 조건 및 기대 결과를 기술 합니다.
> 이 문서와 목업화면을 바탕으로 기획 의도를 파악해서 프론트 화면과 기능을 구현해야 합니다.

## 개요

결재함은 AI Agent가 제출한 결재 요청을 운영자가 확인하고 승인·거부하는 화면이다.

Agent는 전략 채택·제거, 봇 생성·전략 배정·전략 변경·중지·재개·삭제, 예산 변경, 리스크 규칙 변경 등 운영에 영향을 미치는 작업을 수행하기 전에 결재 요청을 생성한다. 운영자는 결재함에서 요청 내역과 근거 자료를 검토한 뒤 최종 결정을 내린다.

### A-1: 결재 이력 조회

**기획 의도**: 운영자가 Agent의 결재 요청 전체를 한눈에 파악한다. 테이블 형태로 유형, 제목, 요청자, 요청일, 상태를 표시한다.

**화면 구성** — 레이아웃은 [목업](../mockups/approvals.html) 참조
- 필터 바: 상태 필터 (A-2) + 유형 필터 (A-3)
- 테이블: 결재 목록
- 페이지네이션 (A-4)

**테이블 필드**

| 필드 | 설명 | key |
|------|------|-----|
| 유형 뱃지 | 결재 유형별 컬러 뱃지 | `approval.type` |
| 제목 | 결재 요청 제목 | `approval.title` |
| 요청자 | Agent 이름 + ID 병기 | `approval.requester_name`, `approval.requester_id` |
| 요청일 | YYYY-MM-DD HH:mm | `approval.created_at` |
| 상태 뱃지 | 대기/승인/거부 | `approval.status` |
| 처리일 | YYYY-MM-DD HH:mm, 미처리 시 `-` | `approval.resolved_at` |

**인터랙션**
- 행 클릭: 해당 결재 상세 페이지로 이동

---

### A-2: 상태별 필터링

**기획 의도**: 운영자가 처리해야 할 대기 건을 빠르게 찾을 수 있도록 상태 필터를 제공한다.

**필터 조건**

| 필터 | 유형 | 옵션 |
|------|------|------|
| 상태 | 탭 버튼 | 전체 · 대기 · 승인 · 거부 |

**동작 규칙**
- 필터 초기값은 "전체"이다.
- 필터 결과가 0건이면 빈 상태 메시지를 표시한다.

---

### A-3: 유형별 필터링 및 제목 검색

**기획 의도**: 결재 유형별로 분류하여 관심 있는 유형만 조회하거나, 제목 키워드로 특정 결재를 빠르게 찾을 수 있다.

**필터 조건**

| 필터 | 유형 | 옵션 |
|------|------|------|
| 유형 | 셀렉트 | 전체 · 전략 채택 · 전략 제거 · 봇 생성 · 전략 배정 · 전략 변경 · 봇 중지 · 봇 재개 · 봇 삭제 · 예산 변경 · 규칙 변경 |
| 제목 검색 | 텍스트 입력 | 제목 키워드 검색 |

---

### A-4: 페이지네이션

**기획 의도**: 결재 건이 많아질 때 탐색이 가능하도록 오프셋 기반 페이지네이션을 제공한다.

**동작 규칙**
- 페이지당 최대 15행 표시
- "N건 중 M-K" 형식으로 현재 위치 표시
- 이전/다음 버튼 제공

---

### A-5: 결재 정보 및 검토 의견

**기획 의도**: 결재의 메타 정보와 자동 검증 결과를 한눈에 파악한다.

**화면 구성** — 레이아웃은 목업 참조: [대기](../mockups/approval-detail-strategy-adopt.html) · [승인](../mockups/approval-detail-bot-stop.html) · [거부](../mockups/approval-detail-rule-change.html)
- 2열 레이아웃: 좌측 결재 정보, 우측 검토 의견 테이블

**결재 정보 필드**

| 필드 | 설명 | key |
|------|------|-----|
| 요청자 | Agent 이름 + ID 병기 | `approval.requester_name`, `approval.requester_id` |
| 유형 | 결재 유형 뱃지 | `approval.type` |
| 상태 | 대기/승인/거부 뱃지 | `approval.status` |
| 요청일 | YYYY-MM-DD HH:mm | `approval.created_at` |
| 만료일 | 대기 상태에서만 표시 | `approval.expires_at` |
| 처리일 | 승인/거부 상태에서만 표시 | `approval.resolved_at` |
| 처리자 | 승인/거부 상태에서만 표시 | `approval.resolved_by` |
| 참조 리포트 | 전략 채택 시 리포트 링크 (선택) | `approval.report_id` |

**검토 의견 필드**

| 필드 | 설명 | key |
|------|------|-----|
| 검증 주체 | rule_engine, agent:ID 등 | `review.reviewer` |
| 결과 | pass/warn/fail 뱃지 | `review.result` |
| 상세 | 검증 상세 메시지 | `review.detail` |
| 검증 시각 | YYYY-MM-DD HH:mm | `review.created_at` |

---

### A-6: 요청 본문

**기획 의도**: Agent가 작성한 요청 근거를 마크다운 렌더링으로 표시한다.

**동작 규칙**
- 본문은 마크다운으로 작성됨 → HTML로 렌더링하여 표시
- 지원 문법: 제목, 볼드/이탤릭, 목록, 인용문, 코드 블록, 표, 링크, 수평선 (마크다운 레퍼런스 참조)

| 필드 | 설명 | key |
|------|------|-----|
| 본문 | Agent가 작성한 마크다운 텍스트 | `approval.body` |

---

### A-8: 결재 액션 (승인/거부)

**기획 의도**: 운영자가 메모를 남기고 최종 결정을 내린다. 실수 방지를 위해 확인 모달을 제공한다.

**동작 규칙**
- 대기 상태에서만 상단에 승인/거부 버튼 노출
- 승인/거부 버튼 클릭 → 확인 모달 → 메모 입력(선택) → 확정
- 검토 의견에 warn/fail이 있으면 모달에 경고 배너 표시
- 확정 후 상태가 즉시 반영되고 목록으로 돌아감
- 승인/거부 완료 상태에서는 버튼 숨김, 결과 배너 표시

| 필드 | 유형 | 설명 |
|------|------|------|
| 메모 | 텍스트 입력 | 결재 사유 또는 코멘트 (선택) |

---

### A-9: 감사 이력

**기획 의도**: 결재 요청의 생성부터 처리까지 모든 이벤트를 시간순으로 기록하여 투명한 의사결정 추적을 보장한다.

**이벤트 필드**

| 필드 | 설명 | key |
|------|------|-----|
| 시각 | 이벤트 발생 시각 | `audit.created_at` |
| 주체 | 이벤트 발생 주체 (agent:ID, rule_engine, owner 등) | `audit.actor` |
| 내용 | 이벤트 요약 (결재 요청 생성, 검토 의견 완료, 승인/거부 등) | `audit.message` |

**동작 규칙**
- 최신 이벤트가 위에 표시 (역순)
- 주체별 색상 구분: Agent(primary), rule_engine(muted), 승인(positive), 거부(negative)
- 거부 시 사유가 이벤트 내용에 인용으로 포함

---

### A-10: 참조 리포트 (전략 채택)

**기획 의도**: 전략 채택 결재에 첨부된 백테스트 리포트를 별도 페이지에서 상세 확인한다.

**화면 구성** — 레이아웃은 [목업](../mockups/report-detail.html) 참조

**동작 규칙**
- 결재 정보의 "참조 리포트" 링크 클릭 시 리포트 상세 페이지로 이동
- 리포트 페이지에서 백테스트 성과 지표, 자산 곡선 차트, 전략 요약, 리스크 분석을 확인

| 필드 | 설명 | key |
|------|------|-----|
| 백테스트 성과 | 샤프 비율, MDD, 승률, 수익 팩터 등 | `report.metrics.*` |
| 자산 곡선 | 에쿼티 커브 차트 | `report.equity_curve` |
| 전략 요약 | Agent 분석 요약 + 근거 | `report.summary`, `report.rationale` |
| 리스크 분석 | 위험 요소 + 권장 사항 | `report.risks`, `report.recommendation` |

---

### A-11: 승인 완료 상태

**기획 의도**: 승인된 결재를 열람할 때 결과를 명확하게 표시한다.

**화면 구성** — 레이아웃은 [목업](../mockups/approval-detail-bot-stop.html) 참조

**동작 규칙**
- 승인 배너 표시 (positive 색상)
- 결재 정보에 처리일, 처리자 필드 추가
- 액션 버튼(승인/거부) 숨김
- 감사 이력에 승인 이벤트 포함

---

### A-12: 거부 완료 상태

**기획 의도**: 거부된 결재를 열람할 때 거부 사유를 명확하게 표시한다.

**화면 구성** — 레이아웃은 [목업](../mockups/approval-detail-rule-change.html) 참조

**동작 규칙**
- 거부 사유 배너 표시 (negative 색상, 거부 사유 포함)
- 결재 정보에 처리일, 처리자 필드 추가
- 액션 버튼(승인/거부) 숨김
- 감사 이력에 거부 이벤트 + 사유 포함
