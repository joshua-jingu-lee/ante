# 로그인

> ⚠️ 이 문서는 이 화면을 이용하는 사용자의 시나리오와 행동에 따른 제약 조건 및 기대 결과를 기술 합니다.
> 이 문서와 목업화면을 바탕으로 기획 의도를 파악해서 프론트 화면과 기능을 구현해야 합니다.

## 개요

로그인은 운영자 인증을 통해 시스템 접근을 통제하는 화면이다. 최소한의 정보만 노출하여 보안을 유지한다.

## 유저스토리

| ID | 스토리 | 완료 조건 |
|----|--------|----------|
| L-1 | 운영자가 ID와 패스워드를 입력하여 로그인한다 | 인증 성공 시 자금 관리 화면으로 이동, 세션 쿠키 발급 |
| L-2 | 잘못된 인증 정보 입력 시 에러를 확인한다 | "ID 또는 패스워드가 올바르지 않습니다" 표시, 구체적 사유 비노출 |
| L-3 | 세션 만료 시 자동으로 로그인 화면으로 돌아온다 | 401 응답 시 `/login`으로 리다이렉트 |

---

### L-1: 로그인

**기획 의도**: 운영자가 Member ID와 패스워드를 입력하여 시스템에 인증한다.

**화면 구성** — 화면 중앙에 로그인 카드 (400px)

**필드**

| # | 필드 | 설명 | 목업 예시 |
|---|------|------|-----------|
| 1 | 로고 | Ante 로고 이미지 (A, N, T, E) | — |
| 2 | 부제 | 시스템 설명 | `Ante Up. Agents Do the Rest.` |
| 3 | Member ID | 텍스트 입력 | `owner` |
| 4 | 패스워드 | 패스워드 입력 + 표시/숨김 토글 | `••••••••` + 👁 버튼 |
| 5 | 로그인 버튼 | Primary 버튼, 전체 너비 | `로그인` |

**동작 규칙**
- 인증 성공 시 자금 관리 화면(`/treasury`)으로 이동, 세션 쿠키 발급
- 패스워드 표시/숨김 토글: 👁 버튼 클릭 시 `type="password"` ↔ `type="text"` 전환
- Enter 키로 폼 제출 지원

---

### L-2: 인증 실패

**기획 의도**: 잘못된 인증 정보 입력 시 보안을 유지하면서 에러를 안내한다.

**화면 구성** — 폼 상단 에러 메시지 영역

**동작 규칙**
- 인증 실패 시 폼 상단에 에러 메시지 표시: `ID 또는 패스워드가 올바르지 않습니다.`
- ID 불일치/패스워드 불일치를 구분하지 않음 — 구체적 사유 비노출 (보안)
- 에러 배경: `negative-bg`, 텍스트: `negative` 색상
- 정상 상태에서는 에러 메시지 영역 숨김 (`hidden`)

---

### L-3: 세션 만료

**기획 의도**: 세션 만료 시 자동으로 로그인 화면으로 리다이렉트하여 미인증 접근을 방지한다.

**동작 규칙**
- API 응답이 401(Unauthorized)인 경우 `/login`으로 리다이렉트
- 리다이렉트 전 현재 URL을 쿼리 파라미터로 보존 — 재로그인 후 원래 페이지로 복귀 가능
