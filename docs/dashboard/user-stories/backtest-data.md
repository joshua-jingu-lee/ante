# 백테스트 데이터

> ⚠️ 이 문서는 이 화면을 이용하는 사용자의 시나리오와 행동에 따른 제약 조건 및 기대 결과를 기술 합니다.
> 이 문서와 목업화면을 바탕으로 기획 의도를 파악해서 프론트 화면과 기능을 구현해야 합니다.

## 개요

백테스트 데이터는 전략 백테스트에 사용되는 시세(OHLCV)·재무(Fundamental) 데이터의 현황을 파악하고 관리하는 화면이다.

데이터 수집 자체는 Agent 또는 CLI(`ante feed`)가 수행하며, 이 화면은 수집된 데이터의 조회·삭제를 담당한다. 터미널 스타일의 파일 브라우저(왼쪽 목록 + 오른쪽 상세)로 구성하여 데이터셋을 직관적으로 탐색할 수 있게 한다.

| 백테스트 데이터 (Fundamental) | 백테스트 데이터 | [backtest-data-fundamental.html](../mockups/backtest-data-fundamental.html) | `ante data list`, `ante data show` |

## 유저스토리

| ID | 스토리 | 완료 조건 |
|----|--------|----------|
| BD-1 | 운영자가 파일 브라우저에서 디렉토리를 탐색한다 | 한 번에 한 깊이씩 탐색, `../`로 상위 이동, 디렉토리/파일 구분 표시 |
| BD-2 | 운영자가 파일을 선택하면 오른쪽에 상세 정보를 확인한다 | 파일 메타(종목·유형·기간·행 수·용량) + 최근 데이터 미리보기 표시 |
| BD-3 | 운영자가 전체 저장 용량을 파악한다 | 브라우저 상단에 전체 용량 + 유형별 용량 표시 |
| BD-4 | 운영자가 불필요한 데이터셋을 삭제한다 | 삭제 확인 모달 + "사용 중인 백테스트가 있을 수 있습니다" 경고 |
| BD-5 | 운영자가 종목을 빠르게 검색한다 | 브라우저 상단 검색 입력란으로 종목명/코드 필터링 |

### BD-1: 파일 브라우저 탐색

**기획 의도**: 터미널의 디렉토리 탐색 방식으로 데이터셋을 탐색한다. 한 번에 한 깊이만 보여주어 단순하고 직관적인 UX를 제공한다.

**화면 구성** — 레이아웃은 [OHLCV 목업](../mockups/backtest-data.html), [Fundamental 목업](../mockups/backtest-data-fundamental.html) 참조
- 통합 터미널 박스 (단일 배경)
- 상단 행: 검색 입력란 — 전체 너비 (BD-5)
- 하단 행 2열: 왼쪽 파일 브라우저(320px, 경로 + 용량 + 파일 목록) + 오른쪽 상세 패널

**디렉토리 구조**

| 경로 | 설명 |
|------|------|
| `~/data` | 루트 — `ohlcv/`, `fundamental/` 디렉토리 표시 |
| `~/data/ohlcv` | 종목 디렉토리 목록 (종목코드) |
| `~/data/ohlcv/{종목코드}` | 타임프레임별 parquet 파일 목록 (`1d.parquet`, `1h.parquet`, `1m.parquet`) |
| `~/data/fundamental` | 종목 디렉토리 목록 (종목코드) |
| `~/data/fundamental/{종목코드}` | 주기별 parquet 파일 목록 (`quarterly.parquet`, `annual.parquet`) |

**동작 규칙**
- 한 번에 현재 디렉토리의 항목만 표시 (플랫 리스트)
- 첫 항목은 `../` (상위 디렉토리 이동), 루트(`~/data`)에서는 숨김
- 디렉토리 클릭 시 해당 디렉토리로 이동하여 하위 항목 표시
- 파일 클릭 시 오른쪽 상세 패널에 정보 표시 (BD-2)
- 각 파일 항목에 용량·행 수를 메타 정보로 표시

**키보드 탐색**

| 키 | 동작 |
|----|------|
| `↑` / `↓` | 포커스 항목 이동 |
| `Enter` | 디렉토리면 진입, 파일이면 상세 패널 표시 |
| `Backspace` | 상위 디렉토리로 이동 (`../` 동작) |
| `/` | 검색 입력란으로 포커스 이동 |

### BD-2: 데이터셋 상세

**기획 의도**: 선택한 데이터셋의 메타 정보와 최근 데이터를 터미널 출력 형태로 확인한다.

**상세 필드**

| 필드 | 설명 | key |
|------|------|-----|
| File | 파일 경로 | `dataset.path` |
| Symbol | 종목코드 + 종목명 | `dataset.symbol`, `dataset.symbol_name` |
| Type | 데이터 유형 (OHLCV / Fundamental) | `dataset.type` |
| Timeframe | 타임프레임 (1d, 1h, 1m, quarterly, annual) | `dataset.timeframe` |
| Period | 데이터 시작일 ~ 종료일 | `dataset.start_date`, `dataset.end_date` |
| Rows | 데이터 행 수 | `dataset.row_count` |
| Size | 파일 크기 | `dataset.size` |

**최근 데이터 미리보기**
- OHLCV: date, open, high, low, close, volume (최근 5건)
- Fundamental: quarter, revenue, op_profit, net_profit, eps, roe (최근 5건)

**동작 규칙**
- 상세 패널은 `ante data show {path}` CLI 출력을 그대로 재현하는 터미널 스타일
- 파일 미선택 초기 상태에서는 프롬프트(`$`)와 커서만 표시
- 파일 클릭 시 상세 패널 갱신

---

### BD-3: 저장 용량 현황

**기획 의도**: 전체 데이터 용량과 유형별 분포를 파악한다.

**필드**

| 필드 | 설명 | key |
|------|------|-----|
| 전체 용량 | 모든 데이터셋 합계 | `storage.total_size` |
| OHLCV 용량 | 시세 데이터 합계 | `storage.ohlcv_size` |
| Fundamental 용량 | 재무 데이터 합계 | `storage.fundamental_size` |

**동작 규칙**
- 상단 행에서 검색 입력란 오른쪽 끝에 한 줄로 표시: "전체 N GB — OHLCV M MB · Fundamental K MB"
- 현재 탐색 위치와 무관하게 항상 전체 용량 표시

---

### BD-4: 데이터셋 삭제

**기획 의도**: 불필요한 데이터셋을 정리한다. 실수 방지를 위해 확인 모달과 경고를 제공한다.

**동작 규칙**
- 상세 패널에서 삭제 트리거 제공 (구현 시 UI 방식 결정)
- 삭제 확인 모달 표시
- 모달에 데이터셋 정보 요약 (종목, 타임프레임, 기간, 행 수)
- 경고 배너: "이 데이터를 사용 중인 백테스트가 있을 수 있습니다"
- 확정 후 파일 목록에서 해당 항목 제거

---

### BD-5: 종목 검색

**기획 의도**: 데이터셋이 많아질 때 원하는 종목을 빠르게 찾을 수 있도록 한다.

**필드**

| 필드 | 유형 | 설명 |
|------|------|------|
| 검색어 | 텍스트 입력 | 종목명 또는 종목코드로 필터링 |

**동작 규칙**
- `/` 키 또는 검색란 클릭으로 포커스 진입
- 키 입력 시 실시간 필터링 (debounce 200ms)
- 검색 범위: 현재 디렉토리가 아닌 **전체 데이터셋** 대상 — 종목명 또는 종목코드에 검색어가 포함된 항목만 표시
- 검색 결과는 `{유형}/{종목코드}/{파일명}` 형태의 플랫 리스트로 표시 (디렉토리 계층 무시)
- 결과 항목 클릭 시 해당 파일의 디렉토리로 이동하고 상세 패널에 정보 표시
- `Escape` 키로 검색 해제 — 검색어 초기화 후 이전 디렉토리 위치로 복귀, 포커스를 파일 리스트로 이동
- 검색어가 비어 있으면 전체 표시 (현재 디렉토리 기준)
