# 봇 관리

> ⚠️ 이 문서는 이 화면을 이용하는 사용자의 시나리오와 행동에 따른 제약 조건 및 기대 결과를 기술 합니다.
> 이 문서와 목업화면을 바탕으로 기획 의도를 파악해서 프론트 화면과 기능을 구현해야 합니다.

## 개요

봇은 전략을 실행하는 인프라다. 운영자가 봇의 생성·시작·중지·삭제를 제어하고, 실행 상태를 모니터링한다. 투자 성과는 전략의 영역이며 봇 화면에서는 다루지 않는다.

## 화면

| 화면 | 페이지 타이틀 | 목업 | 참고 CLI |
|------|--------------|------|----------|
| 봇 목록 | 봇 관리 | [bots.html](../mockups/bots.html) | `ante bot list` |
| 봇 상세 (실행중) | 봇 상세 | [bot-detail.html](../mockups/bot-detail.html) | `ante bot show {id}` |
| 봇 상세 (중지) | 봇 상세 | [bot-detail-stopped.html](../mockups/bot-detail-stopped.html) | `ante bot show {id}` |

## 유저스토리

| ID | 스토리 | 완료 조건 |
|----|--------|----------|
| B-1 | 운영자가 모든 봇의 현황을 파악한다 | 카드 그리드로 봇 표시, 상태순 정렬 (실행중 → 중지됨 → 오류) |
| B-2 | 운영자가 새 봇을 생성한다 | 생성 모달에서 필수 정보 입력 후 봇 생성 |
| B-3 | 운영자가 봇의 상세 정보를 확인한다 | 상세 페이지에서 운용 전략·실행 설정·예산·보유 종목·실행 로그 표시 |
| B-4 | 운영자가 봇을 시작하거나 중지한다 | 상세 페이지에서 시작·중지 버튼, 중지 시 보유 종목 경고 |
| B-5 | 운영자가 중지된 봇의 설정을 수정한다 | 중지 상태에서만 설정 수정 모달 노출 |
| B-6 | 운영자가 봇을 삭제한다 | 상세 페이지에서 중지 상태일 때만 삭제 버튼 노출, 보유 종목 시 처리 방법 선택 |

### B-1: 봇 목록

**기획 의도**: 모든 봇의 상태를 한눈에 파악한다.

**화면 구성** — 레이아웃은 [목업](../mockups/bots.html) 참조
- 모의투자 배너: `account.trading_mode === 'virtual'` 일 때 info 배너 표시 — "모의 거래 중입니다. 실제 주문이 체결되지 않습니다."
- 상단: "봇 생성" 버튼 (좌측 정렬)
- 카드 그리드: `grid-template-columns: repeat(auto-fill, minmax(340px, 1fr))`

**카드 구성**

카드 좌측에 프로필 영역(아이콘 + 상태 뱃지 세로 중앙정렬), 우측에 봇 정보.

| 필드 | 설명 | key |
|------|------|-----|
| 아이콘 | 봇 상태에 따라 시각 표현 (실행: 글로우 애니메이션, 중지: 회색, 오류: 빨간 배경) | `bot.status` |
| 상태 뱃지 | 아이콘 아래 중앙 배치. 실행중 / 중지됨 / 오류 | `bot.status` |
| 봇 이름 | 표시명 | `bot.name` |
| 봇 ID | 고유 식별자, 상세 페이지 링크 | `bot.id` |
| 전략 | 탑재된 전략명, 전략 상세 링크 | `bot.strategy_name` |
| 실행 간격 | 초 단위 | `bot.interval` |

**동작 규칙**
- 카드에는 액션 버튼을 두지 않음 — 시작·중지·삭제는 상세 페이지에서만 수행
- 정렬 순서: 실행중 → 중지됨 → 오류
- 봇 ID 클릭 시 상세 페이지로 이동

### B-2: 봇 생성

**기획 의도**: 새로운 봇을 생성하여 전략을 실행할 인프라를 준비한다.

**생성 필드**

| 필드 | 유형 | 설명 | key |
|------|------|------|-----|
| Bot ID | 텍스트 입력 | 영문·숫자·하이픈, 고유값 | `bot.id` |
| 이름 | 텍스트 입력 | 대시보드 표시명 | `bot.name` |
| 전략 선택 | 셀렉트 | ADOPTED 상태 전략만 표시 (`strategy_id` 기준, name+version) | `bot.strategy_id` |
| 실행 간격 | 숫자 입력 | 최소 10초 ~ 최대 3,600초 | `bot.interval` |
| 배정예산 | 숫자 입력 | 잔여예산 표시 | `bot.budget` |

**동작 규칙**
- 목록 페이지 "봇 생성" 버튼 클릭 시 모달 표시
- 배정예산 힌트에 잔여예산 표시
- 생성 완료 시 목록에 추가 (초기 상태: 중지)
- 봇에 전략 배정 시 전략 파일을 복제(snapshot)하여 봇이 독립 관리한다. 이후 원본 전략의 상태 변경(ARCHIVED 등)은 운영 중인 봇에 영향을 주지 않는다

### B-3: 봇 상세

**기획 의도**: 선택한 봇의 운용 전략, 실행 설정, 예산, 보유 종목, 실행 로그를 한 페이지에서 확인한다.

**화면 구성** — 레이아웃은 [실행중 목업](../mockups/bot-detail.html), [중지 목업](../mockups/bot-detail-stopped.html) 참조
- 상단: 봇 이름 + ID + 상태별 액션 버튼
- 3열 카드: 운용 전략 / 실행 설정 / 예산
- 보유 종목 테이블
- 실행 로그

**운용 전략 카드**

| 필드 | 설명 | key |
|------|------|-----|
| 전략명 | 전략 상세 링크 | `bot.strategy_name` |
| 버전 | 전략 버전 | `bot.strategy_version` |
| 작성자 | 전략 작성 에이전트 (이름 + ID) | `bot.strategy_author_name`, `bot.strategy_author_id` |
| 설명 | 전략 요약 | `bot.strategy_description` |

**실행 설정 카드**

| # | 필드 | 설명 | key | 목업 예시 |
|---|------|------|-----|-----------|
| 1 | 상태 | 실행중 / 중지 뱃지 | `bot.status` | `실행중` (positive) |
| 2 | 실행 간격 | 초 단위 | `bot.config.interval_seconds` | `60초` |
| 3 | 자동 재시작 | 오류 시 자동 재시작 여부 (ON/OFF 뱃지) | `bot.config.auto_restart` | `ON` (positive) |
| 4 | 최대 재시작 | 최대 재시작 시도 횟수 | `bot.config.max_restart_attempts` | `3회` |
| 5 | 재시작 쿨다운 ? | 재시작 시도 사이의 대기 시간 (초) | `bot.config.restart_cooldown_seconds` | `60초` |
| 6 | 스텝 타임아웃 ? | 전략의 1회 실행(on_step)이 이 시간을 초과하면 경고 (초) | `bot.config.step_timeout_seconds` | `30초` |
| 7 | 스텝당 최대 시그널 ? | 전략의 1회 실행에서 발생할 수 있는 매매 신호 수 상한 | `bot.config.max_signals_per_step` | `50` |

**예산 카드**

| 필드 | 설명 | key |
|------|------|-----|
| 배정금액 | 봇에 할당된 총 예산 | `bot.budget` |
| 매수금액 | 보유 종목 매수 합계 | `bot.invested` |
| 체결대기 | 미체결 주문 금액 | `bot.pending` |
| 잔여예산 | 배정금액 - 매수금액 - 체결대기 | `bot.available` |

**보유 종목 테이블**

| 컬럼 | 설명 | key |
|------|------|-----|
| 종목 | 종목명 + 종목코드 | `position.symbol` |
| 수량 | 보유 수량 | `position.qty` |
| 평균단가 | 매수 평균가 | `position.avg_price` |
| 현재가 | 현재 시장가 | `position.current_price` |
| 매수금액 | 수량 x 평균단가 | `position.cost` |
| 평가금액 | 수량 x 현재가 | `position.value` |
| 미실현 손익 | 평가금액 - 매수금액 | `position.unrealized_pnl` |
| 수익률 | 미실현 손익 / 매수금액 | `position.return_pct` |
| 실현 손익 | 이미 청산된 손익 | `position.realized_pnl` |

**실행 로그**

| 필드 | 설명 | key |
|------|------|-----|
| 시각 | 실행 시각 | `log.timestamp` |
| 결과 | 성공 / 실패 / 중지 뱃지 | `log.result` |
| 내용 | 실행 내역 설명 | `log.message` |

**동작 규칙**
- 실행 설정 카드에서 `?` 표시된 필드는 hover 시 툴팁으로 설명 표시 (`.hint-icon` + `data-hint`)
- 헤더에서 "전체 보기" 링크로 전체 로그 페이지 이동
- 상세 페이지에는 최근 로그 10건만 표시
- 손익 양수는 positive 색상, 음수는 negative 색상
- 수량 0인 종목은 실현 손익만 표시 (나머지 "—")

### B-4: 봇 시작/중지

**기획 의도**: 봇의 실행 상태를 제어한다. 중지 시 보유 종목이 있으면 경고한다.

**시작 동작**
- 상세 페이지에서 "시작" 버튼 클릭
- 즉시 실행 상태로 전환

**중지 모달 — 보유 종목 없음**

| 필드 | 설명 | key |
|------|------|-----|
| Bot | 봇 이름 + ID | `bot.name`, `bot.id` |
| 보유 종목 | "없음" | `bot.position_count` |

**중지 모달 — 보유 종목 있음**

| 필드 | 설명 | key |
|------|------|-----|
| Bot | 봇 이름 + ID | `bot.name`, `bot.id` |
| 보유 종목 | N종목 (종목명 나열) | `bot.positions` |
| 체결대기 | 미체결 주문 금액 | `bot.pending` |

**동작 규칙**
- 중지 시 현재 실행 중인 전략 사이클 완료 후 중지
- 미체결 주문은 취소되지 않음
- 보유 종목 있을 시 경고 배너: "보유 종목 N개가 유지됩니다. 중지 후 포지션을 직접 관리해야 합니다"

---

### B-5: 봇 설정 수정

**기획 의도**: 중지된 봇의 설정을 변경하여 전략 교체, 예산 조정 등을 수행한다.

**수정 필드**

| # | 필드 | 유형 | 수정 가능 | key | 설명 |
|---|------|------|-----------|-----|------|
| 1 | Bot ID | 텍스트 | 불가 (읽기 전용) | `bot.id` | — |
| 2 | 이름 | 텍스트 입력 | 가능 | `bot.name` | — |
| 3 | 전략 선택 | 셀렉트 | 가능 | `bot.strategy_id` | active 상태 전략만 표시 |
| 4 | 실행 간격 (초) | 숫자 입력 | 가능 | `bot.config.interval_seconds` | 10~3,600 |
| 5 | 배정예산 | 숫자 입력 | 가능 | `bot.budget` | — |
| 6 | 자동 재시작 | 토글 | 가능 | `bot.config.auto_restart` | ON/OFF |
| 7 | 최대 재시작 횟수 | 숫자 입력 | 가능 | `bot.config.max_restart_attempts` | 1~10 |
| 8 | 재시작 쿨다운 (초) ? | 숫자 입력 | 가능 | `bot.config.restart_cooldown_seconds` | 10~600 |
| 9 | 스텝 타임아웃 (초) ? | 숫자 입력 | 가능 | `bot.config.step_timeout_seconds` | 5~120 |
| 10 | 스텝당 최대 시그널 ? | 숫자 입력 | 가능 | `bot.config.max_signals_per_step` | 1~200 |

**동작 규칙**
- 중지 상태에서만 "설정 수정" 버튼 노출
- Bot ID는 disabled 상태로 변경 불가
- 배정예산 힌트에 현재 배정예산 + 최대 가능 금액 표시

### B-6: 봇 삭제

**기획 의도**: 불필요한 봇을 제거한다. 보유 종목이 있으면 처리 방법을 선택하도록 한다.

**삭제 모달 — 보유 종목 없음**

| 필드 | 설명 | key |
|------|------|-----|
| Bot | 봇 이름 + ID | `bot.name`, `bot.id` |
| 보유 종목 | "없음" | `bot.position_count` |

**삭제 모달 — 보유 종목 있음**

| 필드 | 설명 | key |
|------|------|-----|
| Bot | 봇 이름 + ID | `bot.name`, `bot.id` |
| 보유 종목 | N종목 | `bot.position_count` |

포지션 처리 옵션:

| 옵션 | 설명 |
|------|------|
| 강제 청산 후 삭제 | 보유 종목을 전량 시장가 매도한 후 Bot 삭제 (`handle_positions=liquidate`) |
| 포지션 유지 후 삭제 | 보유 종목을 유지한 채 Bot만 삭제. 포지션은 운영자가 직접 관리 (`handle_positions=keep`) |

**동작 규칙**
- 상세 페이지에서 중지 상태일 때만 삭제 버튼 노출 (실행 중에는 삭제 불가)
- 삭제 시 거래 이력과 성과 기록은 보존
- 보유 종목 없으면 즉시 삭제 확인
- 보유 종목 있으면 처리 방법 라디오 선택 필수
