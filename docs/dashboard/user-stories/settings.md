# 설정

> ⚠️ 이 문서는 이 화면을 이용하는 사용자의 시나리오와 행동에 따른 제약 조건 및 기대 결과를 기술 합니다.
> 이 문서와 목업화면을 바탕으로 기획 의도를 파악해서 프론트 화면과 기능을 구현해야 합니다.

## 개요

설정은 시스템 운영에 필요한 항목을 모듈별로 구분하여 관리하는 화면이다. 좌측 열에 계좌·알림·시스템 설정을, 우측 열에 리스크 규칙을 배치하여 운용 설정과 안전 규칙을 한 화면에서 조망한다.

계좌·리스크 규칙은 헤더에서 선택된 계좌 컨텍스트에 종속된다. 알림·시스템은 전역 설정이다.

## 유저스토리

| ID | 스토리 | 모듈 | 완료 조건 |
|----|--------|------|----------|
| ST-1 | 운영자가 시스템 거래 상태를 확인하고 비상 정지/재개한다 | Account | ACTIVE/HALTED 상태 표시, Kill Switch + 확인 모달 |
| ST-2 | 운영자가 현재 계좌의 거래소 연결 정보를 확인한다 | Account | 브로커, 모드, 계좌번호, 거래소, 통화, 수수료율 표시 |
| ST-3 | 운영자가 알림 채널과 수준을 설정한다 | Notification | 텔레그램 수준, 체결 알림, 일일 리포트 토글 |
| ST-4 | 운영자가 시스템 전역 표시 설정을 변경한다 | System | 금액 단위 위치, 로그 수준 선택 |
| ST-5 | 운영자가 계좌의 리스크 규칙 파라미터를 조정한다 | Rule Engine | 계좌 룰 + 전략별 룰 기본값 편집, 개별 저장 |

---

### ST-1: 시스템 상태 (Account)

**기획 의도**: 현재 거래 상태를 즉시 파악하고, 비상 시 모든 계좌의 거래를 일괄 정지할 수 있는 Kill Switch를 제공한다.

**화면 구성** — 좌측 열 1번째 카드

**ACTIVE 상태**

| # | 필드 | 설명 | 목업 예시 |
|---|------|------|-----------|
| 1 | 안내 메시지 | 정상 운용 상태 안내 (muted) | `현재 모든 봇의 거래가 정상 운용 중입니다.` |
| 2 | 비상 거래 정지 버튼 | Negative 스타일 버튼 | `비상 거래 정지` |

**HALTED 상태**

| # | 필드 | 설명 | 목업 예시 |
|---|------|------|-----------|
| 1 | 정지 메시지 | 거래 정지 상태 (negative, bold) | `거래가 정지되었습니다` |
| 2 | 정지 정보 | 정지 시각 + 사유 (muted) | `정지 시각: 2026-03-15 09:45 · 사유: 긴급 시장 변동` |
| 3 | 거래 재개 버튼 | Positive 스타일 버튼 | `거래 재개` |

**비상 거래 정지 모달**

| # | 필드 | 설명 | 목업 예시 |
|---|------|------|-----------|
| 1 | 모달 제목 | Negative 색상 | `비상 거래 정지` |
| 2 | 안내 문구 | 정지 영향 범위 설명 | `모든 봇의 거래가 즉시 중단됩니다.` `실행 중인 사이클이 완료된 후 정지되며, 보유 포지션은 유지됩니다.` |
| 3 | 정지 사유 | 텍스트 입력 (필수) | `예: 긴급 시장 변동` |
| 4 | 경고 | Warning 배경 알림 | `⚠ 이 작업은 실행 중인 모든 봇에 영향을 줍니다.` |
| 5 | 취소 버튼 | Outline 스타일 | `취소` |
| 6 | 확인 버튼 | Negative 스타일 | `거래 정지 확인` |

**동작 규칙**
- **비상 거래 정지**: `POST /api/system/halt` — 모든 ACTIVE 계좌를 SUSPENDED로 전환 (`AccountService.suspend_all`)
- **거래 재개**: `POST /api/system/activate` — 모든 SUSPENDED 계좌를 ACTIVE로 복구 (`AccountService.activate_all`). DELETED 계좌는 대상 제외
- 정지 시 정지 사유(`reason`)와 정지 요청자(`suspended_by`: 현재 로그인 멤버)를 함께 전송
- 상태 전환 후 카드 표시가 ACTIVE ↔ HALTED로 즉시 전환

> **참조**: Kill Switch는 Account.status 기반으로 동작한다. [account.md](../../specs/account/account.md) Kill Switch 통합 섹션 참조.

### ST-2: 거래소 연결 (Account)

**기획 의도**: 현재 선택된 계좌의 거래소 연결 정보를 표시하여 어떤 브로커·모드·비용 구조로 거래 중인지 즉시 파악하게 한다.

**화면 구성** — 좌측 열 2번째 카드

**필드**

| # | 필드 | 설명 | key | 목업 예시 |
|---|------|------|-----|-----------|
| 1 | 브로커 | 브로커 표시 이름 | `account.name` | `한국투자증권` |
| 2 | 거래 모드 | 뱃지 표시 | `account.trading_mode` | `실전` (primary) |
| 3 | 계좌번호 | 마스킹 표시 (mono) | `account.credentials.account_no` | `5001****-01` |
| 4 | 거래소 | 거래소 코드 | `account.exchange` | `KRX` |
| 5 | 통화 | 통화 코드 | `account.currency` | `KRW` |
| 6 | 매수 수수료율 | 퍼센트 표시 | `account.buy_commission_rate` | `0.015%` |
| 7 | 매도 수수료율 | 퍼센트 표시 | `account.sell_commission_rate` | `0.195%` |

**거래 모드 뱃지 매핑**

| TradingMode | 라벨 | 뱃지 variant |
|-------------|------|-------------|
| VIRTUAL | 가상 | warning |
| LIVE | 실전 | primary |

**동작 규칙**
- 계좌번호는 API 응답에서 마스킹된 값을 표시 — `GET /api/accounts/:id`
- 수수료율은 소수값을 퍼센트로 변환하여 표시 (× 100) — 예: `0.00015` → `0.015%`
- 헤더에서 계좌를 전환하면 이 카드의 정보도 함께 갱신
- 이 카드는 읽기 전용 — 계좌 정보 수정은 CLI(`ante account set-credentials`)로만 가능

### ST-3: 알림 (Notification)

**기획 의도**: 텔레그램 알림 채널의 수준과 개별 알림 항목의 활성화 여부를 설정한다. 계좌에 종속되지 않는 전역 설정이다.

**화면 구성** — 좌측 열 3번째 카드

**필드**

| # | 설정 | 설명 | 유형 | key | 목업 예시 |
|---|------|------|------|-----|-----------|
| 1 | 텔레그램 알림 수준 | 텔레그램 알림 발송 기준 | 셀렉트 | `notification.telegram_level` | `important` |
| 2 | 체결 알림 | 매수/매도 체결 시 즉시 알림 | 토글 | `notification.on_fill` | ON |
| 3 | 일일 리포트 | 매일 장 마감 후 수익 요약 리포트 | 토글 | `notification.daily_report` | OFF |

**텔레그램 알림 수준**

| 값 | 설명 |
|----|------|
| `all` | 모든 이벤트 알림 |
| `important` | 거래 체결, 룰 위반, 봇 상태 변경 |
| `critical` | Kill Switch, 계좌 정지, 시스템 장애 |
| `off` | 알림 비활성화 |

**동작 규칙**
- 변경 즉시 자동 저장 (별도 저장 버튼 없음)
- 저장 API: `PUT /api/config/{key}` (body: `{ "value": "...", "category": "..." }`)
- 토글: `active` 클래스로 ON/OFF 상태 표현

### ST-4: 시스템 (System)

**기획 의도**: 시스템 전역 표시 옵션과 로그 수준을 설정한다. 계좌에 종속되지 않는 전역 설정이다.

**화면 구성** — 좌측 열 4번째 카드

**필드**

| # | 설정 | 설명 | 유형 | key | 목업 예시 |
|---|------|------|------|-----|-----------|
| 1 | 금액 단위 위치 | 통화 단위(원)의 표시 위치 | 라디오 버튼 2개 | `display.currency_position` | `₩ 1,000,000` / `1,000,000 원` |
| 2 | 시스템 로그 수준 | 로그 출력 상세도 | 셀렉트 | `system.log_level` | `INFO` |

**동작 규칙**
- 변경 즉시 자동 저장 (별도 저장 버튼 없음)
- 저장 API: `PUT /api/config/{key}` (body: `{ "value": "...", "category": "..." }`)
- 금액 단위 위치: 선택한 옵션에 `btn-primary` 스타일 적용, 미선택은 `btn-outline`

### ST-5: 리스크 규칙 (Rule Engine)

**기획 의도**: 현재 선택된 계좌의 리스크 규칙 파라미터를 테이블 형태로 편집할 수 있게 한다. 계좌 룰과 전략별 룰 기본값을 섹션 구분자로 분리하여 계층 구조를 시각적으로 표현한다.

**화면 구성** — 우측 열 전체, 테이블 레이아웃 (규칙 | 값 | 저장)

**계좌 룰 (Account Rules)**

| # | 규칙 | 설명(하위 텍스트) | key | 기본값 | 목업 예시 |
|---|------|-------------------|-----|--------|-----------|
| 1 | 일일 손실 한도 (%) | 일일 손실이 이 비율 초과 시 추가 매수 차단 | `DailyLossLimitRule.max_daily_loss_percent` | 5.0 | `5.0` |
| 2 | 총 노출 한도 (%) | 총 자산 대비 최대 포지션 노출 비율 | `TotalExposureLimitRule.max_exposure_percent` | 80.0 | `80.0` |
| 3 | 거래 허용 시간 | Account 설정에서 자동 주입 | Account `trading_hours_start`~`trading_hours_end` | — | `09:00–15:30` (읽기전용) |

**전략별 룰 기본값 (Strategy Rules)**

| # | 규칙 | 설명(하위 텍스트) | key | 기본값 | 목업 예시 |
|---|------|-------------------|-----|--------|-----------|
| 4 | 종목당 최대 비중 (%) | 봇 배정 예산 대비 단일 종목 최대 비중 | `PositionSizeRule.max_position_percent` | 10.0 | `30.0` |
| 5 | 미실현 손실 한도 (%) | 배정 예산 대비 미실현 손실 초과 시 추가 매수 차단 | `UnrealizedLossLimitRule.max_unrealized_loss_percent` | 10.0 | `10.0` |
| 6 | 시간당 최대 거래 수 | 봇당 시간당 거래 횟수 제한 | `TradeFrequencyRule.max_trades_per_hour` | 10 | `10` |

**동작 규칙**
- 테이블 내에 `계좌 룰` / `전략별 룰 기본값` 섹션 구분자(소제목 행)로 계층 구분
- 각 행마다 개별 `저장` 버튼 — 값 변경 후 저장 클릭 시 해당 룰 파라미터만 업데이트
- 저장 API: `PUT /api/accounts/:id/rules/:rule_type` (body: `{ "config": { "key": value } }`)
- 거래 허용 시간은 **읽기 전용** — Account의 `trading_hours_start`/`trading_hours_end`에서 자동 주입. 값 대신 텍스트 표시, 저장 버튼 대신 "읽기전용" 라벨
- 전략별 룰 기본값: 여기서 설정한 값은 해당 계좌에 새로 배정되는 봇의 기본 룰 파라미터로 적용. 개별 봇의 룰은 봇 상세 화면에서 오버라이드 가능
- 값 검증: 숫자 필드는 양수만 허용, 퍼센트 필드는 0~100 범위
- 설정 변경 시 `ConfigChangedEvent`가 발행되어 RuleEngine이 실시간으로 룰 파라미터를 갱신

> **참조**: 룰 목록 및 파라미터 상세는 [rule-engine.md](../../specs/rule-engine/rule-engine.md) 구체적 룰 목록 섹션 참조.
