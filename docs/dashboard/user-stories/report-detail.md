# 리포트 상세

> ⚠️ 이 문서는 이 화면을 이용하는 사용자의 시나리오와 행동에 따른 제약 조건 및 기대 결과를 기술 합니다.
> 이 문서와 목업화면을 바탕으로 기획 의도를 파악해서 프론트 화면과 기능을 구현해야 합니다.

## 개요

리포트 상세는 AI Agent가 작성한 전략 검증 리포트의 전체 내용을 열람하는 읽기 전용 화면이다.

Agent는 전략 백테스트 후 리포트를 제출하며, 운영자는 리포트에서 백테스트 성과 지표·자산 곡선 차트·전략 요약·리스크 분석을 확인하고 전략 채택 여부를 판단한다. 결재 상세(전략 채택)의 참조 리포트 링크를 통해 진입하거나, 전략과 성과 목록에서 직접 접근한다.

### R-1: 리포트 정보

**기획 의도**: 리포트의 메타 정보를 1행 좌측 카드에 표시하여 어떤 전략의 어떤 버전 리포트인지 즉시 파악하게 한다.

**화면 구성** — 1행 좌측(3열 중 1열) 카드

**필드**

| # | 필드 | 설명 | key | 목업 예시 |
|---|------|------|-----|-----------|
| 1 | 리포트 ID | 리포트 고유 ID (mono) | `report.report_id` | `report-20260313-001` |
| 2 | 전략명 | 전략 이름 (mono) | `report.strategy_name` | `momentum_breakout_v2` |
| 3 | 버전 | 전략 버전 | `report.strategy_version` | `v2.0` |
| 4 | 상태 | 리포트 상태 뱃지 | `report.status` | `제출됨` (warning) |
| 5 | 수행자 | Agent 이름 + ID 병기, 링크 | `report.submitted_by_name`, `report.submitted_by` | 전략 리서치 1호 `strategy-dev-01` |
| 6 | 수행일 | YYYY-MM-DD HH:mm | `report.submitted_at` | `2026-03-13 09:20` |

**상태 뱃지 매핑**

| ReportStatus | 라벨 | 뱃지 variant |
|-------------|------|-------------|
| DRAFT | 초안 | muted |
| SUBMITTED | 제출됨 | warning |
| REVIEWED | 검토됨 | info |
| ADOPTED | 채택됨 | positive |
| REJECTED | 미채택 | negative |
| ARCHIVED | 보관됨 | muted |

### R-2: 백테스트 설정

**기획 의도**: 백테스트가 어떤 조건으로 실행되었는지(초기 자금, 타임프레임, 매수/매도 수수료율, 슬리피지율)와 어떤 데이터를 사용했는지(대상 종목)를 한 카드에 표시하여 결과의 재현 조건을 즉시 파악하게 한다.

**화면 구성** — 1행 중앙(3열 중 2열) 카드

**필드**

| # | 필드 | 설명 | key | 목업 예시 |
|---|------|------|-----|-----------|
| 1 | 백테스트 기간 | 시작~종료 (기간 표시) | `report.backtest_period` | `2024-01 ~ 2026-03 (2년)` |
| 2 | 초기 자금 | 통화 포맷 | `detail_json.config.initial_balance` | `₩10,000,000` |
| 3 | 타임프레임 | 봉 주기 | `detail_json.config.timeframe` | `1d` |
| 4 | 매수 수수료율 | 퍼센트 표시 | `detail_json.config.buy_commission_rate` | `0.015%` |
| 5 | 매도 수수료율 | 퍼센트 표시 | `detail_json.config.sell_commission_rate` | `0.195%` |
| 6 | 슬리피지율 | 퍼센트 표시 | `detail_json.config.slippage_rate` | `0.1%` |
| 7 | 대상 종목 | 종목 코드 축약 표시 | `detail_json.datasets[].symbol` | `005930`, `000660` 외 8 |

**대상 종목 축약 규칙**
- 종목 수 3개 이하: 전부 표시 (mono) — 예: `005930`, `000660`, `035420`
- 종목 수 4개 이상: 앞 2개 표시 + "외 N" — 예: `005930`, `000660` 외 8
- 종목 수 0개 또는 datasets 없음: "-" 표시

**동작 규칙**
- `detail_json.config`와 `detail_json.datasets`는 BacktestConfig/DatasetInfo 직렬화 필드. 에픽 #728 구현 후 사용 가능
- 기존 리포트에 해당 키가 없으면 "설정 정보 없음" 안내 표시
- 수수료율은 소수값을 퍼센트로 변환하여 표시 (× 100) — 예: `0.00015` → `0.015%`

> **참조**: key 상세 구조는 [backtest.md](../../specs/backtest/backtest.md) BacktestConfig, DatasetInfo 섹션 및 [report-store.md](../../specs/report-store/report-store.md) detail_json 내부 구조 참조.

### R-3: 백테스트 성과

**기획 의도**: 백테스트의 핵심 성과 지표를 1행 우측 카드에 표시하여 전략의 수익성과 리스크를 한눈에 평가하게 한다.

**화면 구성** — 1행 우측(3열 중 3열) 카드

**필드**

| # | 필드 | 설명 | key | 색상 | 목업 예시 |
|---|------|------|-----|------|-----------|
| 1 | 누적 수익률 | 수익률(%) + 금액 병기 | `report.total_return_pct`, `detail_json.initial_balance`, `detail_json.final_balance` | positive/negative | `+42.3% (+4,230,000원)` |
| 2 | 총 거래 수 | 거래 횟수 | `report.total_trades` | — | `127회` |
| 3 | 승률 | 백분율 | `report.win_rate` | — | `62.2%` |
| 4 | 수익 팩터 | 총이익/총손실 비율 | `detail_json.metrics.profit_factor` | — | `1.85` |
| 5 | 샤프 비율 | 위험 대비 초과수익 | `report.sharpe_ratio` | — | `1.72` |
| 6 | 최대 낙폭 | 낙폭(%) + 금액 병기 | `report.max_drawdown_pct`, `detail_json.equity_curve[]` | negative | `-6.8% (-680,000원)` |
| 7 | 평균 수익 | 수익률(%) + 금액 병기 | `detail_json.metrics.avg_profit`, `detail_json.initial_balance` | positive | `+1.12% (+78,500원)` |
| 8 | 평균 손실 | 수익률(%) + 금액 병기 | `detail_json.metrics.avg_loss`, `detail_json.initial_balance` | negative | `-0.60% (-42,300원)` |
| 9 | 수수료 합계 | 전체 수수료 | `detail_json.metrics.total_commission` | muted | `58,200원` |
| 10 | 활성 거래일 | 거래 발생 일수 | `detail_json.trades[]` 고유 날짜 수 산출 | — | `89일` |

**산출 규칙**
- **누적 수익률 금액**: `final_balance - initial_balance`
- **최대 낙폭 금액**: `equity_curve[]`에서 peak - trough (고점 대비 최대 하락 금액)
- **평균 수익 수익률**: `avg_profit / initial_balance × 100`
- **평균 손실 수익률**: `avg_loss / initial_balance × 100`

> **참조**: `detail_json`은 BacktestResult를 JSON 직렬화한 필드. metrics 키 목록은 [backtest.md](../../specs/backtest/backtest.md) 성과 지표 계산 참조

### R-4: 자산 곡선

**기획 의도**: 백테스트 기간 동안의 자산 변화를 듀얼 축 차트로 시각화하여 전략의 성과 추이와 일별 변동성을 직관적으로 파악하게 한다. treasury 자산추이와 동일한 듀얼 축 구성으로 시각적 일관성을 유지한다.

**화면 구성** — 2행 전폭(1열) 카드

| 영역 | 설명 | key |
|------|------|-----|
| 기간 손익 요약 | 차트 상단 — 총 손익 금액 + 수익률 | `detail_json.initial_balance`, `detail_json.final_balance`, `report.total_return_pct` |
| 듀얼 축 차트 | 높이 240px | `detail_json.equity_curve[]` |

**데이터 구조**

| 필드 | 설명 | key |
|------|------|-----|
| 시점 | 타임스탬프 | `equity_curve[].timestamp` |
| 자산 | 평가 자산 (포지션 포함) | `equity_curve[].equity` |
| 잔고 | 현금 잔고 | `equity_curve[].balance` |
| 일별 수익률 | 프론트 산출 — `(equity[n] - equity[n-1]) / equity[n-1] × 100` | 프론트 산출 |

**동작 규칙**
- 듀얼 축: 좌축 — 총 자산 추이 (Area), 우축 — 일별 수익률 (Bar)
- X축: 기간, 좌측 Y축: 금액(원), 우측 Y축: 수익률(%)
- 줌/패닝: 드래그로 구간 확대, 스크롤로 이동
- 기간 손익 요약은 현재 뷰포트 기준으로 동적 재계산 — 뷰포트 시작점 equity와 끝점 equity로 `(end - start) / start × 100` 산출
- 초기 상태: 전체 백테스트 기간 표시 — 예: `기간 손익 +4,230,000원 · 기간 수익률 +42.3%`
- 손익 금액·수익률에 positive/negative 색상 적용

> **참조**: treasury 자산추이 차트와 동일한 시각적 구성 ([treasury.html](../mockups/treasury.html))
> **참조**: equity_curve 리샘플링은 백엔드에서 처리 (#741). 프론트는 API 응답 데이터를 그대로 렌더링.

### R-5: 전략 요약 · 리스크 분석

**기획 의도**: Agent가 작성한 전략 분석과 리스크 평가를 2열 레이아웃으로 나란히 표시하여 운영자가 전략의 장단점을 비교 검토하게 한다.

**화면 구성** — 3행 2열(detail-grid) 레이아웃

**전략 요약 (좌측 카드)**

| # | 필드 | 설명 | key |
|---|------|------|-----|
| 1 | 요약 | Agent가 작성한 전략 개요 | `report.summary` |
| 2 | 근거 | 전략 수립 근거 | `report.rationale` |

**리스크 분석 (우측 카드)**

| # | 필드 | 설명 | key |
|---|------|------|-----|
| 1 | 위험 요소 | 식별된 리스크 | `report.risks` |
| 2 | 권장 사항 | 운용 권장 사항 | `report.recommendations` |

**동작 규칙**
- 텍스트는 마크다운으로 작성됨 → HTML로 렌더링하여 표시
- 각 필드는 소제목(strong) + 본문 구조
