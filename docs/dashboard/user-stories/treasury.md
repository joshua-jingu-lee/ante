# 자금 관리

> ⚠️ 이 문서는 이 화면을 이용하는 사용자의 시나리오와 행동에 따른 제약 조건 및 기대 결과를 기술 합니다.
> 이 문서와 목업화면을 바탕으로 기획 의도를 파악해서 프론트 화면과 기능을 구현해야 합니다.

## 개요

자금 관리는 운영자가 **지금 얼마 벌고 있나**와 **돈이 어디에 있나**를 파악하는 화면이다.
계좌 잔고, Ante 관리 자금 현황과 자산 추이 차트, 봇별 예산 배분, 자금 변동 이력을 제공한다.

**목업**: [treasury.html](../mockups/treasury.html) · [treasury-history.html](../mockups/treasury-history.html)

## 유저스토리

| ID | 스토리 | 완료 조건 |
|----|--------|----------|
| T-1 | 운영자가 계좌 잔고와 Ante 관리 자금 현황을 확인한다 | 계좌 잔고 카드(총 자산 평가, 총 손익, 예수금, 매수가능금액) + Ante 관리 자금 카드(관리자산 평가·손익, 배정예산, 잔여예산, 보유종목 평가·손익, 체결대기) 표시 |
| T-2 | 운영자가 자산 추이와 일별 수익률을 기간별로 조회한다 | 1주/1개월/3개월/1년/전체 선택 → 듀얼 축 차트(자산 Area + 수익률 Bar) 갱신 + 기간 손익·수익률 표시 |
| T-3 | 운영자가 봇별 예산 비중을 시각적으로 파악한다 | 파이 차트로 봇별 배분 비율 표시 |
| T-4 | 운영자가 봇별 예산 상세를 테이블로 확인한다 | 봇별 예산 테이블 (배정예산, 사용예산, 잔여예산, 체결대기, 보유종목 평가금액·손익, 종목 수익률, 예산 수익률) |
| T-5 | 운영자가 특정 봇의 예산을 변경한다 | 봇 선택 → 금액 입력 → 변경 확인 모달 → 실행 |
| T-6 | 운영자가 최근 자금 변동 이력을 확인한다 | 자금 변동 테이블 (시각, 유형, Bot, 금액, 비고) + 전체 보기 링크 |
| T-7 | 운영자가 자금 변동 전체 이력을 조회한다 | 거래 이력 페이지에서 필터·페이지네이션 포함 로그 조회 |

### 공통: 가상 거래 배너

**기획 의도**: 가상 거래 계좌(`account.trading_mode === 'virtual'`)일 때 사용자가 실제 자산과 혼동하지 않도록 안내한다.

**동작 규칙**
- `trading_mode === 'virtual'`일 때만 main 영역 최상단에 info 배너 표시
- 문구: "가상 거래 계좌입니다. 표시된 손익은 시뮬레이션 결과입니다."
- 실전투자 계좌에서는 배너를 표시하지 않는다

### T-1: 계좌 잔고 · Ante 관리 자금

**기획 의도**: 운영자가 자금 관리 화면 진입 시 거래소 계좌 전체 잔고와 Ante 관리 범위의 자금 현황을 즉시 확인한다.

**화면 구성** — 레이아웃은 [목업](../mockups/treasury.html) 참조

#### 계좌 잔고 카드

카드 헤더: 계좌 이름 + 계좌번호 + 수수료·매도세·동기화 시각

| 필드 | 설명 |
|------|------|
| 총 자산 평가 | 거래소 계좌 전체 자산 평가액 |
| 총 손익 | 거래소 계좌 전체 손익 (금액 + 수익률) |
| 예수금 | 거래소 예수금 |
| 매수가능금액 | 실제 매수 가능 금액 |

#### Ante 관리 자금 카드

카드 헤더: "Ante 관리 자금" + 운용 중 Bot 수 + 경고 메시지(해당 시)

**상단 행 (4열)**

| 필드 | 설명 |
|------|------|
| Ante 관리자산 평가 | Ante가 관리하는 자산의 평가 금액 |
| Ante 관리자산 손익 | Ante 관리 자산 기준 손익 (금액 + 수익률) |
| Bot 배정예산 | 봇에 배정된 총 예산 |
| 잔여예산 | 배정 후 남은 예산 |

**하단 행 (3열)**

| 필드 | 설명 |
|------|------|
| 보유종목 평가금액 | Ante 관리 종목의 현재 평가 금액 |
| 보유종목 손익 | 보유종목 기준 손익 (금액 + 수익률) |
| 체결대기 자금 | 미체결 주문에 묶인 금액 |

**동작 규칙**
- 손익 양수는 positive 색상, 음수는 negative 색상
- 손익 금액 앞에 부호 표시 (+/-)
- 수익률은 괄호 안에 % 표시
- 잔여예산이 매수가능금액을 초과하면 카드 헤더에 warning 배지 표시

**Virtual 모드 (`trading_mode === 'virtual'`) 표시 규칙**

Virtual 모드에서는 브로커 API를 사용하지 않으므로 데이터 소스와 표시 항목이 달라진다. 스펙: [treasury.md Virtual 모드 자산 평가 동기화](../../specs/treasury/treasury.md#virtual-모드-자산-평가-동기화-d-trs-01)

| 구분 | Live | Virtual |
|------|------|---------|
| 계좌 잔고 카드 | 브로커 API 조회 결과 표시 | 표시하지 않음 — 브로커 동기화가 없으므로 의미 없는 값 |
| Ante 관리자산 평가·손익 | 브로커 API 기반 `_eval_amount` | Trade DB(`PositionHistory`) + DataProvider 시세 기반 |
| 보유종목 평가금액·손익 | 브로커 API 기반 | Trade DB 포지션 × DataProvider 현재가 |
| 예수금·매수가능금액 | 브로커 API 조회 | 표시하지 않음 |

- Virtual 모드에서는 계좌 잔고 카드 자체를 숨기고, Ante 관리 자금 카드만 표시한다
- Ante 관리 자금의 평가·손익 산식은 모드와 무관하게 동일하다 (`get_summary()` 하위 로직 공통)
- 가상 거래 배너가 최상단에 표시된다 (공통 섹션 참조)

> 데이터 소스: Live — KIS API 계좌 잔고 조회 + Treasury 모듈 예산 데이터 / Virtual — Trade DB + DataProvider + Treasury 모듈 예산 데이터

### T-2: 자산 추이 차트

**기획 의도**: 운영자가 기간별 자산 변동 추이와 일별 수익률 변화를 시각적으로 파악한다.

**화면 구성** — Ante 관리 자금 카드 내부 하단에 위치 (border-top으로 구분)

- 좌측: "자산 추이" 타이틀 + 기간 손익·기간 수익률
- 우측: 기간 선택 버튼 그룹
- **좌축 (Area Chart)**: 총 자산 추이 (`total_asset`) — 자산 규모 변화
- **우축 (Bar Chart)**: 일별 수익률 (`daily_return`) — 일별 성과 변화
- 수익률 바: 양수는 positive 색상, 음수는 negative 색상
- 차트 하단 우측: 산정 기준일 (muted, 11px)

**기간 선택**

| 버튼 | 설명 | key |
|------|------|-----|
| 1주 | 최근 1주 추이 | `period=1w` |
| 1개월 | 최근 1개월 추이 | `period=1m` |
| 3개월 | 최근 3개월 추이 | `period=3m` |
| 1년 | 최근 1년 추이 | `period=1y` |
| 전체 | 전체 기간 추이 | `period=all` |

**데이터 조회**

프론트엔드는 기간 버튼에 따라 `start_date`/`end_date`를 계산하여 동일한 API를 호출한다. 백엔드는 일별 스냅샷만 반환하며, 주/월 단위 집계는 하지 않는다.

| 버튼 | API 호출 | 예상 행 수 |
|------|---------|-----------|
| 1주 | `start_date=7일전, end_date=오늘` | ~7 |
| 1개월 | `start_date=30일전, end_date=오늘` | ~30 |
| 3개월 | `start_date=90일전, end_date=오늘` | ~90 |
| 1년 | `start_date=365일전, end_date=오늘` | ~365 |
| 전체 | `start_date` 생략 | 최대 ~1,825 (5년) |

> API: `GET /api/treasury/snapshots?account_id={id}&start_date={}&end_date={}`
> 데이터 소스: `daily_snapshots` 테이블 — 스펙: [treasury.md 일별 자산 스냅샷](../../specs/treasury/treasury.md#일별-자산-스냅샷-daily-asset-snapshot)

**차트 데이터 매핑**

| 시리즈 | 축 | 차트 유형 | 데이터 key | 색상 |
|--------|---|----------|-----------|------|
| 총 자산 | 좌축 (원) | Area | `snapshot.total_asset` | 기본 테마 색상 |
| 일별 수익률 | 우축 (%) | Bar | `snapshot.daily_return × 100` | 양수: positive, 음수: negative |

**동작 규칙**
- 기본 선택: 1주
- 기간 버튼 클릭 시 `start_date`/`end_date` 재계산 → API 호출 → 차트 갱신
- 차트 X축: 반환된 `snapshot_date`
- 차트 좌축: `total_asset` (원 단위, Area Chart)
- 차트 우축: `daily_return` (% 단위, Bar Chart)
- 기간 손익: 선택 기간 내 `daily_pnl`의 합계로 프론트엔드에서 산출
- 기간 수익률: 선택 기간 내 `daily_return`의 기하평균 `(1+r₁)×(1+r₂)×…×(1+rₙ) - 1`으로 프론트엔드에서 산출
- 호버 시 해당 날짜의 총 자산 + 일별 수익률 툴팁 표시
- 산정 기준일: 가장 최근 스냅샷의 `snapshot_date`, 차트 하단 우측에 muted 텍스트로 표시

### T-4: Bot당 예산 테이블

**기획 의도**: 운영자가 봇별 예산 상세를 숫자로 확인한다.

**화면 구성** — 전체 폭 카드

| 컬럼 | 설명 |
|------|------|
| Bot ID | 봇 식별자 (봇 상세 링크) |
| 배정예산 | 해당 봇에 배정된 예산 |
| 사용예산 | 매수에 사용된 금액 |
| 잔여예산 | 배정예산 - 사용예산 - 체결대기 |
| 체결대기 | 미체결 주문에 묶인 금액 |
| 보유종목 평가금액 | 보유 종목 현재 평가액 |
| 보유종목 손익 | 보유 종목 손익 (positive/negative 색상) |
| 종목 수익률 | 보유종목 손익 / 사용예산 × 100 |
| 예산 수익률 | 보유종목 손익 / 배정예산 × 100 |

**동작 규칙**
- 손익·수익률 양수는 positive, 음수는 negative 색상
- Bot ID 클릭 시 봇 상세 페이지로 이동

### T-5: Bot 예산 관리

**기획 의도**: 운영자가 특정 봇의 예산을 변경한다.

**화면 구성** — detail-grid 우측 카드

- Bot 선택 드롭다운 (봇 이름 + 상태 표시)
- 현재 배정예산 표시
- 변경할 예산 입력
- "예산 변경" 버튼

**동작 규칙**
- 예산 변경은 Bot이 중지된 상태에서만 가능 — **프론트엔드 + 백엔드 이중 방어**
  - **프론트엔드**: 운용 중인 봇 선택 시 변경 버튼 비활성화 + "봇이 중지된 상태에서만 예산을 변경할 수 있습니다" 안내 텍스트
  - **백엔드**: 실행 중인 봇에 대한 예산 변경 요청 시 `409 Conflict` 반환
- "예산 변경" 클릭 → 확인 모달 표시 (봇 이름, 현재 예산, 변경 예산, 차액)
- 모달에서 "변경 확인" 클릭 → API 호출 → 성공 시 관련 카드 갱신

### T-6: 최근 자금 변동

**기획 의도**: 운영자가 최근 자금 변동 내역을 빠르게 확인한다.

**화면 구성** — 전체 폭 카드, 헤더에 "전체 보기 →" 링크

| 컬럼 | 설명 |
|------|------|
| 시각 | 변동 발생 시각 (`YYYY-MM-DD HH:mm`, mono) |
| 유형 | 할당(positive 뱃지) / 회수 / 체결(primary 뱃지) |
| Bot | 대상 봇 |
| 금액 | 변동 금액 (양수: positive, 음수: negative) |
| 비고 | 부가 설명 (muted) |

**동작 규칙**
- 최근 N건만 표시 (기본 5~10건)
- "전체 보기" 클릭 시 treasury-history.html로 이동

### T-7: 자금 변동 전체 이력

**기획 의도**: 운영자가 자금 변동 전체 이력을 필터링하여 조회한다.

**화면 구성** — [treasury-history.html](../mockups/treasury-history.html) 참조

**동작 규칙**
- T-6과 동일한 테이블 구조 + 필터(기간, 유형, 봇) + 페이지네이션
- "전체 보기" 링크에서 진입
