# 에이전트 관리

> ⚠️ 이 문서는 이 화면을 이용하는 사용자의 시나리오와 행동에 따른 제약 조건 및 기대 결과를 기술 합니다.
> 이 문서와 목업화면을 바탕으로 기획 의도를 파악해서 프론트 화면과 기능을 구현해야 합니다.

## 개요

에이전트 메뉴는 운영자가 Ante의 AI Agent를 등록·조회·관리하기 위해 사용한다.

Agent는 API 토큰으로 인증하며, 소속(org)과 scope로 활동 범위가 제한된다. 이 화면에서는 Agent의 등록, 상태 전환, 토큰 관리, 권한 범위 설정을 수행할 수 있다.

## 유저스토리

### 에이전트 목록

| ID | 스토리 | 완료 조건 |
|----|--------|----------|
| M-1 | 운영자가 active/suspended 상태의 Agent를 카드 형태로 조회한다 | 각 카드에 프로필(이모지 아바타), ID, 이름, 상태 뱃지, 소속, 마지막 활동 시각이 포함. revoked 에이전트는 목록에 표시하지 않는다 |
| M-2 | 운영자가 새 Agent를 등록하고 토큰을 발급받는다 | "+ 에이전트 등록" 버튼 → 등록 모달(Agent ID, 이름, 소속, 권한 범위 체크박스) → 등록 완료 시 토큰 표시 모달(1회 표시, 복사 버튼, 닫으면 재확인 불가 경고) |
| M-3 | 운영자가 Agent를 일시 정지·재활성화·삭제한다 | **상세 페이지에서만** 상태 전환 버튼 제공. active: "일시 정지" + "삭제". suspended: "재활성화" + "삭제" |

### 에이전트 상세

| ID | 스토리 | 완료 조건 |
|----|--------|----------|
| M-4 | 운영자가 Agent의 기본 정보와 활동 이력을 확인한다 | 기본 정보(ID, 이름, 소속, 상태) + 활동 정보(생성일, 생성자, 마지막 활동, 일시 정지 시각) 표시 |
| M-5 | 운영자가 Agent의 토큰을 재발급한다 | "토큰 재발급" 버튼 → "기존 토큰이 즉시 무효화됩니다" 경고 표시 → 새 토큰 1회 표시 |
| M-6 | 운영자가 Agent의 정보를 수정한다 | "정보 수정" 버튼 → 수정 모달(이름, 소속, 권한 범위) → 저장 시 즉시 반영 |

---

### M-1: 에이전트 목록 조회

**기획 의도**: 운영자가 active/suspended 상태의 Agent를 한 화면에서 파악한다. 삭제(revoked)된 에이전트는 목록에 노출하지 않는다.

**화면 구성**
- 우측에 "+ 에이전트 등록" 버튼
- 카드 그리드: `repeat(auto-fill, minmax(340px, 1fr))`, gap 16px

**카드 필드** — 레이아웃은 [목업](../mockups/agents.html) 참조

| 필드 | 설명 | key |
|------|------|-----|
| 프로필 아바타 | 이모지 (72×72 원형). positive 계열 배경 | `member.emoji` (미지정 시 🤖) |
| ID | 15px semibold | `member.member_id` |
| 이름 | 13px text-muted | `member.name` |
| 상태 뱃지 | 헤더 우측에 한국어 라벨로 표시 | `member.status`: `'active'`(활성) · `'suspended'`(일시 정지) |
| 소속 | `badge-muted` 태그 | `member.org` |
| 마지막 활동 | YYYY-MM-DD HH:mm | `member.last_active_at` |
| scope 태그 목록 | 카드 하단에 태그로 표시 | `member.scopes: string[]` |
| 카드 링크 | 전체 카드 클릭으로 상세 페이지 이동 | `member.type === 'agent'` |

**인터랙션**
- 아바타 hover: `scale(1.15) rotate(-8deg)` 트랜지션
- 카드 hover: `border-color: primary`
- suspended 카드: 프로필 아바타 dimmed (`opacity: 0.4`)

---

### M-2: 에이전트 등록 및 토큰 발급

**기획 의도**: 새 Agent를 등록하면서 즉시 API 토큰을 발급받는다. 토큰은 보안상 1회만 표시되므로, 운영자가 반드시 복사하도록 경고와 복사 버튼을 함께 제공한다.

**등록 모달 필드**

| 필드 | 유형 | 설명 |
|------|------|------|
| Agent ID | 텍스트 입력 | 영문, 숫자, 하이픈만 허용. 고유값 |
| 이름 | 텍스트 입력 | 자유 텍스트 (한글 가능) |
| 소속 (org) | 텍스트 입력 | 자유 텍스트. 기본값 `default` |
| 권한 범위 (Scopes) | 체크박스 그룹 | strategy:read, strategy:write, report:write, approval:create, approval:review, bot:read, data:read, data:write, backtest:run |

**완료 조건**
- 등록 성공 시 토큰 표시 모달로 전환
- 토큰 모달에 Agent ID, 발급된 토큰(모노스페이스), 복사 버튼 표시
- ⚠️ "이 토큰은 다시 표시되지 않습니다. 안전한 곳에 복사해 두세요." 경고 배너 표시
- 닫기 후 재확인 불가 — 분실 시 M-6 재발급 필요

---

### M-3: Agent 상태 전환 (일시 정지·재활성화·삭제)

**기획 의도**: Agent의 수명 주기를 관리한다. 일시적으로 정지하거나, 문제 해결 후 재활성화하거나, 더 이상 필요 없는 Agent를 삭제할 수 있다.

**상태 전이 다이어그램**

```
active ──일시 정지──▷ suspended ──재활성화──▷ active
  │                      │
  └──삭제──▷ revoked ◁──삭제──┘
```

**상태별 표현**

| 상태 | 라벨 | 뱃지 | 상세 페이지 버튼 |
|------|------|------|-----------------|
| active | 활성 | `badge-positive` | 정보 수정(outline) · 일시 정지(outline) · 삭제(outline, text-negative) |
| suspended | 일시 정지 | `badge-warning` | 정보 수정(outline) · 재활성화(positive) · 삭제(outline, text-negative) |
| revoked | 삭제됨 | `badge-negative` | — (목록·상세 미노출) |

> 모든 상태 전환·수정 액션은 **상세 페이지에서만** 제공한다. 삭제(revoked)된 에이전트는 목록에 표시하지 않는다.

**동작 규칙**
- 일시 정지: active → suspended. 해당 Agent의 API 토큰이 즉시 비활성화된다.
- 재활성화: suspended → active. 기존 토큰이 복원된다.
- 삭제: active/suspended → revoked. **되돌릴 수 없다.** 토큰 영구 무효화. 삭제된 에이전트는 목록에서 사라진다.
- 삭제 실행 전 확인 모달 필수: "이 작업은 되돌릴 수 없습니다. 정말 삭제하시겠습니까?"

---

### M-4: 에이전트 상세 조회

**기획 의도**: 운영자가 개별 Agent의 전체 정보를 한 화면에서 확인한다. 기본 정보, 활동 이력, 토큰 관리, 권한 범위를 섹션별로 구성한다.

**화면 구성** — 레이아웃은 [목업](../mockups/agent-detail.html) 참조
- 헤더: ← 에이전트 관리 (뒤로가기) + Agent ID
- 상세 헤더: Agent ID, 상태 뱃지, 소속 뱃지, 이름 + 상태 전환 버튼(M-3)
- 2열 그리드: 기본 정보 카드 + 활동 정보 카드
- 토큰 관리 카드 (M-5)
- 권한 범위 표시 + "정보 수정" 버튼 (M-6)

**기본 정보 필드**

| 필드 | 설명 | key |
|------|------|-----|
| Agent ID | 모노스페이스 | `member.member_id` |
| 이름 | 자유 텍스트 | `member.name` |
| 소속 | org 값 | `member.org` |
| 상태 | 뱃지 표시 | `member.status` |

**활동 정보 필드**

| 필드 | 설명 | key |
|------|------|-----|
| 생성일 | YYYY-MM-DD HH:mm | `member.created_at` |
| 생성자 | 등록한 멤버 ID | `member.created_by` |
| 마지막 활동 | YYYY-MM-DD HH:mm | `member.last_active_at` |
| 일시 정지 시각 | 미정지 시 `-` 표시 | `member.suspended_at` |

---

### M-5: 토큰 재발급

**기획 의도**: Agent의 토큰이 노출되었거나 분실된 경우 새 토큰을 발급한다. 기존 토큰은 즉시 무효화되므로 운영자에게 명확한 경고를 제공한다.

**동작 규칙**
- 토큰 관리 섹션에 현재 토큰 접두어 표시: `ante_ak_****`
- ⚠️ "토큰 재발급 시 기존 토큰이 즉시 무효화됩니다." 경고 상시 표시
- "토큰 재발급" 버튼 클릭 → 확인 모달 → 확인 후 새 토큰 1회 표시 (M-2와 동일한 토큰 표시 모달)

---

### M-6: 에이전트 정보 수정

**기획 의도**: Agent의 이름, 소속, 권한 범위를 한 모달에서 수정한다. 등록 모달(M-2)과 유사한 형태이나, Agent ID는 변경 불가(읽기 전용 표시).

**진입점**
- 상세 화면: 상세 헤더의 "정보 수정" 버튼 (active/suspended 상태만)

**수정 모달 필드**

| 필드 | 유형 | 설명 |
|------|------|------|
| Agent ID | 읽기 전용 | 변경 불가. 모노스페이스로 표시 |
| 이름 | 텍스트 입력 | 자유 텍스트 (한글 가능) |
| 소속 (org) | 텍스트 입력 | 자유 텍스트 |
| 권한 범위 (Scopes) | 체크박스 그룹 | 현재 부여된 scope가 체크된 상태로 표시 |

**Scope 목록**

| Scope | 설명 |
|-------|------|
| strategy:read | 전략 조회 |
| strategy:write | 전략 등록·수정 |
| report:write | 리포트 작성·제출 |
| approval:create | 결재 요청 생성 |
| approval:review | 결재 검토·의견 제출 |
| bot:read | 봇 상태 조회 |
| data:read | 시장 데이터·백테스트 데이터 조회 |
| data:write | 데이터 업로드·수정 |
| backtest:run | 백테스트 실행 |

**동작 규칙**
- "저장" 클릭 시 변경 사항 즉시 반영, 해당 Agent의 다음 API 요청부터 적용
- "취소" 클릭 시 모달 닫힘, 변경 사항 버림
- revoked 상태의 Agent는 수정 불가
