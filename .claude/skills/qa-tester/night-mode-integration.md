# /night-mode QA 통합 가이드

야간 자율 모드(`/night-mode`)에서 QA 테스트를 자동 실행하는 절차를 정의한다.

## 통합 위치

night-mode의 "모든 작업 종료 시" 단계 직전에 QA 단계를 삽입한다.

```
... (기존 이슈 순차 구현 흐름)

N. QA 검증 단계 (모든 구현 이슈 완료 후)
   - `/qa-test all --fix` 실행
   - FAIL 발견 시 자동으로 버그 이슈 등록 → 즉시 수정 시도
   - 수정 후 재검증 PASS 시 이슈 자동 close

N+1. 모든 작업 종료 시 (기존 정리 절차)
```

## night-mode 작업 흐름에 추가할 단계

기존 night-mode 흐름의 4번(완료 또는 실패 후 → 1번으로 돌아가 다음 이슈 선택)과
"모든 작업 종료 시" 사이에 다음 단계를 삽입한다:

### QA 검증 단계

```
5. QA 통합 테스트 실행
   - 모든 오픈 이슈 처리 완료 후 (더 이상 진행할 구현 이슈가 없을 때)
   - `/qa-test all --fix` 실행

6. QA 결과 처리
   - PASS: 다음 단계(정리)로 진행
   - FAIL + --fix 성공: 자동 close된 이슈 번호 기록
   - FAIL + --fix 실패: 미해결 버그 이슈 목록을 최종 보고에 포함
   - ERROR: 환경 문제로 판단, 사용자에게 에스컬레이션
```

## QA → 버그 수정 연동 흐름

```
/qa-test all --fix
  ├── Scenario A: PASS → (처리 없음)
  ├── Scenario B: FAIL
  │     ├── gh issue create → #700
  │     ├── /implement-issue #700
  │     ├── Scenario B 재실행
  │     │     ├── PASS → gh issue close #700
  │     │     └── FAIL → #700 열어둠, 최종 보고에 포함
  │     └── (다음 FAIL Scenario로)
  └── Scenario C: PASS → (처리 없음)
```

## night-mode 최종 보고에 포함할 QA 요약

야간 모드 종료 시 완료/실패/스킵 이슈 요약에 QA 결과를 추가한다:

```
## 야간 모드 결과 보고

### 구현 이슈
- 완료: #601, #602, #603
- 실패: #604 (3회 복구 실패)
- 스킵: #605 (선행 의존 미완)

### QA 검증
- 실행: /qa-test all --fix
- 결과: 45/50 PASS, 3 FAIL (자동 수정), 2 FAIL (미해결)
- 자동 수정 완료: #700, #701, #702
- 미해결 버그: #703, #704
```
